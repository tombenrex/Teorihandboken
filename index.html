HTML & CSS

Html

Ett HTML dokument består av olika taggar som ger plats för att skapa innehåll på sidan. Här placerar vi olika element med texter, rubriker, bilder, knappar, formulär för inputs där vi kan skriva in text och mycket mer.

CSS

CSS uppgift är att koppla ihop med HTML elementen för att bestämma vilken färg, storlek, placering de i sin tur ska ha. Användningsområdet är stort men för att beskriva det kort, bestämmer det vilket utseende eller vilka animationer vi vill ha.

Layouttekniker CSS

CSS Framework

Här importeras kod från färdiga bibliotek, ett nytt sätt att skapa. En stor fördel är att man sparar tid, eftersom det är mer effektivt än att skapa från grunden. Nackdel är att man kanske inte använder sig av all kod, vilket i sin tur skapar större filer som innebär längre laddningstider.

Css float property

Något som användes mer förr. Detta fungerar med alla äldre webbläsare och kräver inte lika mycket kod. Nackdelen är det är svårare att få till en bättre konstruktion och centrera innehållet

Css flexbox

En modern teknik som används idag av utvecklare. Det har fördelar som att det både är lätt att förstå och använda. Fördelaktigt när du vill centrera innehållet eller placera det i en rad.

Css grid

Även denna är en modern teknik som gör det till en fördel om du vill skapa helskärms layouter. En mer avancerad metod där man inte behöver lika mycket html kod för att skapa extra elements som exempelvis flexbox. Med detta kan vi placera element i specifika ”grid-areas”.

javascript

Stort användningsområde

Några exempel på vad man kan gör med JS:

  • Hantera användarens input (musklick, tangenttryck, scroll) och koppla dem till funktioner.
  • Validera formulär och beräkna datan.
  • Hämta olika API, importera dessa, skapa i tabeller.
  • Manipulera och byta ut eller skapa nya element i HTML/CSS.
  • Skapa effekter.

xml

samband mellan html & xml

Båda är ”markup språk” vilket innebär att de använder taggar för att definiera element i dokumentet. Skiftlägeskänslig när taggar och attribut anges. Underhålls av W3C (World Wide Web Consoritum).

samband mellan json & xml

Både Json och XML kan användas för att ta emot data från webbservrar. Lättläst text som både människor och datorer förstår. Används stort inom API och är plattformsoberoende.

Tillgänglighet

Genom att ha en tydlig layout med lätt navigering. Användaren ska kunna justera textstorleken eller färgen för lättare läsbarhet. Alternativa texter till bilder för ”skärmuppläsare” som ska kunna tyda allt innehåll som finns.

För de personer som inte kan använda sig av en datormus så ska innehållet kunna navigeras med tangentbordet i stället.

Dos lagen

Lagen om tillgänglighet till digital offentlig service. En lag i Sverige som uppfyller kraven som EU satt. Offentliga men även privata aktörer har denna lag att följa. Allt för att öka tillgängligheten.

Metadata

Google eller andra söktjänster

När vår server ligger uppe med metadatan inkluderad använder sig söktjänster som exempelvis Google av denna data. Som i sin tur för vidare informationen till användaren som gjort en sökning inom området. Google använder datan för att kunna kategorisera innehållet rätt.

Internets konstruktion

Internets hårdvara

Internet är ett globalt nätverk sammankopplade med datorer och servrar som kommunicerar med varandra genom olika protokoll. Den fysiska infrastrukturen består av kablar och fiberoptik som i sin tur ansluts av routrar och switchar. Större webbsidor, appar och molntjänster kräver större serverhallar för att kunna ta emot all trafik och hantera datan.

När vi surfar

En hemsida anropas genom att skriva in en URL i adressfältet. Webbläsaren i sin tur gör en DNS-förfrågan för att hitta rätt IP-adress. När den funnit rätt IP svarar servern med en HTML fil som i sin tur innehåller länkar till CSS, JavaScript och bilder.

Förkortningar

DNS = Omvandlar domännamn till ip-adress
IP & http = Protokoll som styr hur data skickas och tas emot på internet

http-protokoll

förfrågan

Protokollet bygger på förfrågan och svar mellan en klient som exempelvis webbläsaren. När vi anger en adress skickar vi en förfrågan som i sin tur hämtar den informationen vi ber om. Det är cookies, webbläsartyp och URL som förfrågan består av.

svaret

Servern tar emot förfrågan och skickar tillbaka den begärda informationen som exempelvis HTML sidan. Vid lyckad hämtning innehåller den statuskoden 200, om inte 404 Not Found.

http eller https?

Genom att använda sig av HTTPS kan man känna sig säkrare då informationen krypteras. På så sätt kan inte lösenord eller annat läcka. Använder inte hemsidan någon HTTP:s bortprioriteras den från google och oftast kommer en varning upp att hemsidan inte är säker.

Förkortningar

HTTP = Hypertext transfer protocol
HTTPS = Hypertext transfer protocol secure

Front & Backend

Frontend

Det vi ser framför oss när vi använder en hemsida kallas för frontend. Själva designen/layouten som är skapat av elementen. Bilder, knappar menyer och funktionaliteten. HTML, JS och CSS är en del av det och använder sig av http för att kunna hämta datan och visa användaren en hemsida.

Backend

Den kod/data vi inte ser och sker ”bakom kulisserna”. Den har en viktig uppgift att sammanlänka informationen som finns tillgänglig och fånga upp. Det kan exempelvis vara information från användare som angett sina uppgifter vid en inloggning. Den tar emot, bearbetar och skickar tillbaka rätt information genom http till frontend som visar upp innehållet.

Prestandan & påverkan på hemsidor

Mycket funktioner, inte alltid bra

Det som kan påverka prestandan är allt som väger mycket i megabyte eller gigabyte. Stora bilder, mycket kod och funktioner som hemsidor/servrar ska ta ansvar över.

Har domänen/servern hemsidan finns på en dålig uppkoppling påverkas det av längre laddningstider för användaren.


Fungerar allt ovan optimalt som det ska men ändå upplevs en sämre prestanda på hemsidan, skulle det kunna vara ens komponenter i datorn som inte är tillräckliga. Uppdateringar av mjukvaran kan också påverka.

Framework

Olika typer av frameworks har också en påverkan på prestandan. Även fast man inte använder all kod behöver den ändå ladda in det beroende på hur allt är uppbyggt.

Responsiv design

Genom att använda sig av olika tekniker i sitt utförande av kod så anpassar sig layouten och förblir lika användarvänlig på alla skärmar.

CSS

I CSS används flexbox & grid för att skapa kolumner och radstrukturer. Där storleken bestäms att den ska anpassa sig efter skärmen. På så sätt blir den dynamisk och responsiv.

Vi kan även sätta upp speciella regler i något som kallas för media queries. Där kan man exempelvis bestämma att vid 768 pixlar (mindre skärm) och mindre ska något hända. Exempelvis att navigationsmenyn ska tas bort och gömmas under en knapp för att skapa med plats åt innehållet.

Versionshantering & GIT

Vad står GIT för?

GIT har ingen betydelse men utvecklaren Linus Torvalds som skapade det sa:
”Global Information Tracker” – Om du använder det rätt,
”Goddamn Idiotic Tool” – Om du gör något dumt med det.

VCS (Version Control System)

Git är alltså ett versionshanteringssystem som används för att spåra ändringar i kod som skapas. Väldigt bra att använda i större eller mindre projekt när man jobbar tillsammans med andra. Alla ändringar loggas och blir någonting fel kan man gå tillbaka till en tidigare version.
